# 🌟 오늘의 별자리

> 컴퓨터·스마트폰을 많이 쓰는 현대인을 위한 **데일리 손목 웰니스 루틴 웹앱**.
> 짧은 운동 루틴 · 웹캠 기반 손목 체크 · 밤하늘 별자리 수집을 하나로 묶어,
> 바쁜 일상에서도 쉽고 재미있게 손목 건강을 돌보게 합니다.

운동을 하나 끝낼 때마다 **오늘의 별자리에 별이 켜지고**, 6개를 다 하면 별자리가 완성돼
**밤하늘 도감**에 쌓입니다. 우주비행사 고양이 마스코트가 그날 상태에 맞는 한마디를 건넵니다.

설치·회원가입 없이 브라우저에서 바로 실행됩니다. 영상은 저장하지 않고 **좌표·수치만**
로컬(`localStorage`)에 남겨 개인정보를 보호합니다.

> ⚠️ 본 서비스는 의료기기가 아니며 질병의 진단·치료 목적이 아닙니다.
> 통증이 있거나 질환이 의심되면 의사와 상담하세요. 체크값은 참고용 자가 수치입니다.

## ✨ 핵심 기능

| 축 | 설명 | 상태 |
|---|---|---|
| 🎬 **오늘의 루틴** (메인, 매일) | 일러스트 손이 시범을 보이고, 웹캠 인식으로 따라하기 회수를 세어 줍니다. 매일 동일한 풀코스 6종, 1~3분 숏폼 연속 재생. | ✅ 구현 |
| 📏 **손목 체크** (가끔) | 웹캠 + AI 손 인식으로 **굽힘·폄**(옆모습)과 **좌우 편위**(정면) 최대각을 재고 기록·비교합니다. 주 1회 "제안" 칩. | ✅ 구현 |
| 🌌 **밤하늘 도감** | 완성한 별자리를 밤하늘에 모읍니다. 계절별 4종 × 4계절 = **16개**. 탭하면 그날의 일기 카드. | ✅ 구현 |
| 🧭 **적응형 맞춤 루틴** | 체크 기록으로 약한 방향을 조용히 판정해 그 방향 운동의 반복만 소폭 조정합니다. 사용자에겐 긍정 신호만 노출. | ✅ 구현 (6단계 완료) |
| 🎮 **게임** (보조) | 운동 동작을 활용한 미니게임으로 지루함을 덜어 줍니다. | ⏳ 이식 예정 (홈에서 카드 숨김) |

## 🚀 실행 방법

ES modules + 웹캠(`getUserMedia`) + CDN 모듈을 쓰기 때문에 **로컬 웹서버(http)** 로 열어야
합니다. `file://` 로 열면 카메라와 모듈 로딩이 차단됩니다. `localhost` 는 보안 컨텍스트라
카메라가 정상 동작합니다.

### 방법 A — Python (설치된 Python 사용)
```bash
python -m http.server 5500      # 프로젝트 루트에서
# 브라우저에서 http://localhost:5500/ 접속
```

### 방법 B — VS Code Live Server 확장
`index.html` 우클릭 → **Open with Live Server** (기본 `http://127.0.0.1:5500`).

> 최초 접속 시 카메라 권한을 허용하세요. MediaPipe 모델은 CDN에서 내려받으므로
> 인터넷 연결이 필요합니다.

### 테스트 실행 (Node)

빌드 도구·`package.json`·CI 없이, 각 테스트가 **단독 실행 스크립트**입니다.
실패하면 `exit 1`. DOM·카메라가 필요 없는 순수 로직만 검증합니다.

```bash
node js/star.test.mjs         # 오늘의 별자리 한마디 tier 선택
node js/adapt.test.mjs        # 적응형 진행/후퇴(dose) 순수 로직
node js/deviation.test.mjs    # 정면 편위 부호·캡처·저장 스키마
node js/integration.test.mjs  # 측정→focus→dose→긍정신호 이어붙임(seam)
node js/tracking.test.mjs     # delegate GPU→CPU 폴백
```

## 🧭 사용 흐름

```
접속 → 홈: 왼쪽 밤하늘 히어로(고양이 + 오늘의 별자리) / 오른쪽 기능 카드 열
 → 🎬 오늘의 루틴: 원탭 시작 → 시범 따라하기(회수 자동 카운트, 6종 연속 재생)
                  → 운동 하나 끝날 때마다 별 점등 → 컨디션 한 번 탭 → 스트릭 반영
                  → 6/6 완주 시 별자리 완성 연출 + 도감 누적 (중간에 끝내도 그날 인정)
 → 📏 손목 체크: 손 선택(왼/오른) → 옆모습 중립 → 굽힘·폄 캡처
                → 정면 중립 재보정 → 요측·척측(엄지쪽·새끼쪽) 캡처 → 결과
 → 📈 기록: 굽힘/폄 추이 · 편위 합 추이 · 최근 7일 컨디션 · 데일리 루틴 · 운동 기록
 → 🌌 밤하늘 도감: 완성한 별자리 수집(N/16), 탭하면 그날의 일기 카드
```

알림에서 들어오면 `?routine=today` 쿼리로 홈을 건너뛰고 바로 루틴이 시작됩니다.

## 🧩 모듈 설명

### `config.js` — 상수 단일 출처
MediaPipe 모델 URL/옵션, 카메라 설정, 랜드마크 인덱스, 스무딩 프로파일(`live`/`measure`),
보상동작 임계(`COMP_TH`), ROM 판정값, 기능 기준(`FUNCTIONAL_ROM`), 루틴·적응형 파라미터
(`ROUTINE.adaptReps` / `adaptDose` / `adaptImprove`), 한마디 문구 세트(`STAR_MESSAGE`)를
모두 보관합니다.

- MediaPipe CDN은 **버전 고정**(`@0.10.35`) — `@latest` 자동 업데이트로 배포본이 갑자기
  깨지지 않게. 갱신 시 `visionBundle`·`wasmPath` 두 URL을 함께 올립니다.
- 편위 개별 방향 라벨(`DEV_LABEL`)은 **★잠정★** — 실제 손으로 뒤집힌 것이 확인되면
  이 두 값만 맞바꾸면 됩니다(저장 필드명·합 진척률은 손댈 필요 없음).

> 🐛 디버그 플래그(평소 모두 `false`)
> · `DEBUG_GUIDE` — 가이드 인식 진단 로그(`[guide-diag]`/`[flexExt]`)
> · `DEBUG_ADAPT` — 적응형 판정 로그(`[adapt]`/`[star]`)
> · `DEBUG_MEASURE` — 정면 편위 프레임 진단 로그(`[dev]`)

### `tracking.js` — 인식 전담
- `initModels()` — vision_bundle 동적 import → HandLandmarker / PoseLandmarker 로딩
- `createWithFallback()` — delegate **GPU → CPU 순서로 폴백**. 두 모델에 각각 독립 적용해
  GPU 미지원 기기·브라우저에서도 앱이 열립니다
- `startCamera(video)` / `startLoop(onFrame, {pose})` / `stopTracking()` —
  `video.currentTime` 이 바뀔 때만 추론(중복 방지)

### `measurement.js` — 측정 전담 (순수/상태)
- 헬퍼: `norm` / `median` / `ang` / `dist` / `clamp` / `pushN`
- `fingerMetrics(hand)` → `grip`, `spread`, `tipMCP`, `pinch` (손바닥 크기로 정규화)
- `createWristTracker(profile)` — 손목각 → median 버퍼 → 지수평활 → `rel`(중립 기준) →
  `comp`(팔꿈치·어깨 이동량으로 보상동작 감지). `beginNeutral()` / `commitNeutral()`
- `createRomMeasurer()` — 끝범위를 일정 시간 유지하면 좌우 최대각을 자동 저장
- **부호 정규화** — 두 손은 카메라에 거울상으로 잡히므로 A/B가 늘 같은 물리 방향을
  가리키도록 반전합니다. 반전 대상 손이 두 단계에서 서로 반대인 것은 의도된 것:
  | 함수 | 자세 | A (rel<0) | B (rel>0) | 반전하는 손 |
  |---|---|---|---|---|
  | `flexExtRel` | 옆모습 | 굽힘 | 폄 | 오른손 |
  | `deviationRel` | 정면(손바닥이 카메라) | 요측(엄지쪽) | 척측(새끼쪽) | 왼손 |

### `store.js` — 저장·스트릭·별자리·적응형 판정
하나의 루트 키(`wristGarden`) 아래 JSON 트리로 보관하며, `schemaVersion`으로 마이그레이션을
관리합니다(현재 **v2** — 측정 레코드에 편위 필드 추가, 추가 전용이라 옛 기록 무손실).

- **스트릭** — 하루 한 번이라도 활동하면 달성. 하루 공백은 **주 1회 자동 프리즈** 🧊로 유지,
  같은 주 두 번째 공백이나 이틀 이상 공백이면 리셋
- **별자리** — `assignTodayConstellation` / `distributeStars` / `lightStars` /
  `syncStarsToProgress` / `completeTodayConstellation`
- **적응형 판정** — `computeFocus`(약한 방향) · `isRedSignal`(급락) · `isImproving`(개선) ·
  `getAdapt`/`saveAdapt` · `freshComp`(comp 신선도)
- **표시 계산** — `deviationProgress`(편위 합의 기능 진척률)

> ⚠️ `routine` / `reminder` / `sky` / `adapt` 의 기본값은 반드시 `null` — `load()`의 얕은
> 머지가 부분 객체의 내부 키를 보호하지 못하기 때문입니다.

### `routine.js` — 오늘의 루틴 + 적응형 조정 + 한마디
- **매일 동일한 풀코스 6종, 고정 순서** — 로테이션·커스터마이징 없음
  ("고민할 것이 없어야 매일 한다")
- **순한 코스(3종) 자동 전환** — 두 안전 신호 중 하나면 발동: ① 어제 컨디션이 "뻐근해요"
  ② 최근 측정이 직전 대비 급락(red). 둘 다 "더 순하게" 방향이라 안전
- **관대한 판정** — 중간에 끝내도 그날 활동으로 인정. 풀코스 완주(6/6)는 ⭐ 추가 축하일 뿐
- 체크는 7일 주기 **"제안" 칩**만
- `getRoutineGuide()` — 재생 시 dose 조정을 얹은 **사본**을 반환(원본 `GUIDES`는 불변)

### `reminder.js` — 손목 리마인더
- 프리셋(오전 업무 중 10:30 등 4종)/직접 입력으로 **하루 최대 3회**.
  설정 후엔 홈 오른쪽 열 맨 아래의 접힌 칩이 유일한 입구(탭하면 패널이 펼쳐짐)
- 오늘 루틴을 이미 완주했으면 남은 알림은 조용히 스킵. 권한은 한 번만 요청
- ⚠ `setInterval` 기반이라 **탭이 켜져 있을 때만** 동작 — 백그라운드 푸시는
  향후 PWA(Service Worker + Push) 과제

### 가이드 모듈 (`js/guide/`)
- **`guideData.js`** — 운동 = `{id, name, view, steps[]}`. 스텝 타입은
  `intro`(자동) / `follow`(인식 카운트) / `outro`(자동). 애니메이션은
  `anim: [[시간(s), {파라미터}], ...]` 키프레임
- **`guideHand.js`** — 손을 이미지가 아니라 **파라미터 인형**으로 그림.
  `wristAngle`, `devAngle`, `curl`, `curlTip`, `spread`, `pinchGap`.
  `side`(굽힘·폄) / `front`(편위·손가락·핀치) 2뷰
- **`animPlayer.js`** — 매끄러움 3원칙: ① 절대시간 ② `easeInOut` ③ 홀드 구간
- **`stepEngine.js`** — 스텝 진행 + 판정기(detector). 판정기는 `measurement.js` 의
  지표를 재사용:

  | detect 키 | 판정 | 사용 운동 |
  |---|---|---|
  | `flexExt` | `rel` 굽힘·폄 왕복 1세트 = 1회 (폄 목표는 낮게, 2D 한계) | 손목 굽힘·폄 |
  | `deviation` | `rel` 요측·척측 왕복 | 좌우 편위 |
  | `tendonGlide` | `grip`/`tipMCP` 로 쫙→갈고리→주먹 순서 통과 | 힘줄 활주 · 손가락 벌리기 |
  | `pinchHold` | `pinch<0.34` 유지 | 핀치 집기 |
  | `gripHold` | 중립 + `grip<1.2` 유지 | 악력 유지 |

  UX 원칙: 시범과 카운트 **독립**(박자 강요 없음), 목표 각도 **관대**,
  **15초 인식 0 → 탈출구**. 보상동작(comp)은 카운트를 막지 않고 세션 비율만
  집계합니다(적응형 진행 판정의 대리 신호로 사용).

  인식 신뢰 장치:
  - **인식 가시화** — 카메라 PiP에 손 감지 칩(회색/초록, 히스테리시스) +
    관절점 오버레이(거울·크롭 보정), 회수 인정 순간 점 팝 + 테두리 플래시
  - **판정 좌표계** — 가이드는 화면 기준 절대각 왕복 감지(usePose 미사용)라
    왼손/오른손 어느 쪽이든 동일하게 동작(거울상 무관, 실측 검증)
  - **중립 견고화** — 손이 보이는 프레임이 충분히 모여야 기준각 확정

## 🌌 별자리 시스템

앱 이름이자 진행 시각화의 중심. **운동 = 별, 하루 = 별자리, 꾸준함 = 밤하늘**.

### 오늘의 별자리 (홈 히어로)
- 계절(3~5 봄 / 6~8 여름 / 9~11 가을 / 12~2 겨울)에 맞는 4종 중 하나를 **날짜 해시**로
  배정 — 같은 날은 항상 같은 별자리(멱등), 최근 완성한 3개는 가능하면 회피
- 별자리의 별 개수를 6개 운동 슬롯에 나눠 배정(`distributeStars`)하고, 완료한 슬롯의
  배정 합만큼 **그리는 순서대로** 점등. 홈을 나갔다 와도 진행분이 그대로 복원됩니다(멱등)
- 렌더는 `sky.js` 순수 SVG — 안 켜진 별은 흐릿한 밑그림, 켜진 별은 골드 코어 + 헤일로,
  양 끝이 켜진 연결선은 금빛 그라디언트. **직전 렌더 이후 새로 켜진 별만** 반짝입니다
  (상태 복원은 조용히)
- 6/6 완주 → 도감 누적(멱등) + **완성 연출 1회**(샤인 + 이름 배너 3.2초)

### 밤하늘 도감 (`🌌` 화면)
- 화면 전체를 채우는 밤하늘(full-bleed)에 16개 별자리를 `con.pos` 좌표로 흩뿌려 배치
  (가로 Δ0.22·세로 Δ0.20 이상 확보 — 어떤 화면비에서도 안 겹침)
- 완성 = 밝게 + 이름 라벨 / 미완성 = 아주 흐릿 + 잠김. 탭하면 확대 카드
- 완성한 별자리 카드 = **그날의 일기** — 별자리 설명 · 완성한 날 · 그날 루틴(N/6) · 그날 컨디션

### 마스코트 (우주비행사 고양이)
`assets/cat-{idle|happy|sad}.png` 3종. 상태 조건은 말풍선과 동일한 것을 재사용합니다.

| 표정 | 조건 |
|---|---|
| `happy` | 오늘 루틴 완주 |
| `sad` | 이전 활동은 있는데 스트릭이 끊김(며칠 만의 방문) |
| `idle` | 그 외 |

## 🧠 적응형 맞춤 루틴 (측정 기반)

설계: [`docs/맞춤루틴_설계문서_1.md`](docs/맞춤루틴_설계문서_1.md) · 근거: Ryu 1991(기능적 ROM),
증상 유도 점진(response-guided), 24h rule.

**대원칙 — 조용히 맞춤 + 개선 시에만 긍정 신호.** 사용자에게 "약함/부족/나빠짐"을 절대
노출하지 않고, 인식·측정 엔진은 손대지 않은 채 **루틴 파라미터(reps/hold)만** 조정합니다.

| 단계 | 함수 | 규칙 |
|---|---|---|
| 약한 방향 판정 | `computeFocus` | 최근 측정의 낮은 쪽이 기능선(40°) 미만이면 focus, 이상이면 참고 코칭(`focusSoft`). 30일 이상 오래된 측정도 soft |
| 방향 특이적 조정 | `computeDose` | focus 대상 운동의 reps만 +2(soft면 +1). 운동 제거·교체 없음 |
| 진행 (§4.3) | `decideDose` | 무난 3세션 연속 + 직전 대비 악화 없음 + comp ≤15% → **1단계 상승**. reps를 cap(10)까지 소진한 뒤 hold(+3초, 상한 15초) |
| 후퇴 (§4.4) | `decideDose` | stiff → 즉시 원위치(0단계) / 직전 대비 하락 → 1단계 하강. **후퇴가 상승보다 항상 우선** |
| 간격 가드 | `decideDose` | 최근·직전 컨디션이 사흘 이상 벌어지면 비교 부정확 → 유지(`hold-gap`) + streak 리셋 |
| red 클램프 | `computeDose` | 측정 급락한 날은 focus 운동을 **base로 재생**. 상태는 안 건드리므로 다음날 저절로 복귀 |
| 긍정 신호 (§4.5) | `improveSignal` | 개선 + 견딤 streak ≥2 + 오늘 하강 아님 + 3일 간격 + 측정이 14일 이내일 때만 1회 |

적응형은 하루 1회 멱등(`lastAdaptedAt`)이며, 로직을 꺼도 기본 코스로 정상 동작합니다(안전 폴백).

## 💬 오늘의 별자리 한마디

설계: [`docs/보이는_돌봄_설계.md`](docs/보이는_돌봄_설계.md) — **"돌봄은 보이게, 심판은 숨기게."**

내부 적응 신호를 긍정·따뜻한 중립으로 번역해 홈 말풍선에 **하루 한 줄**로 내보냅니다
(`routine.dailyStarMessage`). 판정 로직을 새로 만들지 않고 기존 신호를 읽어 tier만 고르며,
문구는 `config.STAR_MESSAGE` 한 곳에 있습니다.

| 순위 | tier | 조건 |
|---|---|---|
| 1 | `improve` | `improveSignal` 발동 |
| 2 | `gentleStiff` / `gentleRed` | 순한 날 — 자기보고(stiff)는 공감, 앱 추론(red)은 중립 톤 |
| 3 | `levelUp` | 오늘 dose가 올라감 |
| 4 | `focus` | focus 방향 있음 — "약함"이 아니라 "오늘 마음 쓸 방향"으로 |
| 5 | `streak` | 연속 방문 3일↑ 또는 견딤 streak 2↑ |
| 6 | `base` | 기본 — 오늘 별자리 이름을 곁들인 중립·따뜻 |

같은 날 = 같은 문구(날짜 기반 안정 해시, `Math.random` 금지 — 다시 그릴 때마다 바뀌면
"앱이 날 본다"가 아니라 소음이 됩니다). 치환자(`{focus}`/`{guide}`/`{n}`/`{constellation}`)를
채울 수 없는 문구는 후보에서 자동으로 빠집니다.

> **가드레일** — 어떤 문구에도 약함/부족/나빠짐/수치/정상 대비 %/재촉이 들어가면 안 됩니다.

## 📏 손목 체크 (2단계 측정)

```
idle → neutral(옆모습 중립 2.5s) → measure(굽힘·폄 캡처)
     → devNeutral(정면 중립 재보정) → dev(요측·척측 캡처) → result(저장)
```

- 두 단계는 자세도 기준 중립도 달라 **트래커·ROM 측정기를 각각** 씁니다(섞으면 기준각이
  치우침). 편위는 손바닥이 카메라를 보는 자세로 통일 — 손등을 위로 두면 원근 단축으로 측정이 무너집니다
- 끝범위에서 2.2초 유지하면 자동 캡처(흔들리면 타이머 리셋). 측정 중 손 전환 시 이미 캡처된
  A/B 값도 함께 맞바꿔 물리적 의미를 지킵니다
- **편위는 선택 단계** — 못 재도 굽힘·폄은 저장됩니다(레코드에 `null`)
- 결과 표시는 **기능 기준 진척률**만 — "정상인 대비 %"·"N도 남았어요" 같은 부족 표현·처방 금지
  - 굽힘·폄: 각각 40° 기준
  - 편위: 요측/척측 **합** 40° 기준. 합을 쓰는 이유는 ① 근거(Ryu 1991)가 합 기준이고
    ② 합은 요측/척측 라벨이 뒤바뀌어도 값이 같아, 개별 방향 부호가 실제 손으로 확정되기
    전에도 안전한 유일한 축이기 때문입니다
