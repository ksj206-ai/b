# 측정 기반 맞춤 루틴 설계 문서

손목 측정 기록(measurements)을 활용해 매일 루틴을 개인 맞춤으로 조정하는 적응형(adaptive) 시스템.

## 0. 대원칙

**A안 확정: 조용히 맞춤 + 개선 시에만 긍정 신호.**
- 맞춤은 뒤에서 조용히 동작. 사용자에겐 "약하다/부족하다"를 절대 노출 안 함.
- 개선됐을 때만 가끔 긍정 신호("손목이 부드러워지고 있어요 ✨").
- 기존 웰니스 원칙 계승: 부족 표현 금지, 진척 프레이밍, 재촉 금지, A~D 내부 점수 비노출.

**안전 최우선(웰니스 앱, 진단·치료 아님).** 문헌이 "손목 특이적 정밀 용량 규칙은 없다"고 명시 → **보수적·증상 기반** 진행. 정밀 재활 프로토콜처럼 굴지 않는다.

## 1. 근거 (Perplexity 리서치 · PubMed 인용)

| 규칙 | 근거 |
|---|---|
| 기능적 ROM = 굴곡 40°·신전 40° | Ryu et al. 1991 — ADL 최소 기능선. 우리 기존 FUNCTIONAL_ROM과 일치 |
| 40° 미만 = "기능적으로 제한" 1차 플래그 | 문헌 지지. 정상 인구값(굴곡 70~90° 등)은 2차 참고만 |
| 방향 특이적 운동 강화 (굴곡 약하면 굴곡 위주) | 문헌 지지 — 방향 분리 타당 |
| 둘 다 40°↑면 "부족" 아님, 낮은 쪽은 코칭 신호로만 | 문헌 지지 — 비대칭은 2차 신호, 1차 진단 아님 |
| 3세션 잘 견딘 뒤 한 번에 한 변수만 진행 | 반응 유도 점진(response-guided) 원칙 · PT 검수 확정 |
| 다음날 증상이 안전 체크(24h rule) | 문헌 지지 — 다음날 악화면 용량 과다 |
| 통증 시 밀어붙이지 말고 부하 감소 | 문헌 지지 |
| 상한 reps 8~10(단계적), hold 10~15초 | 문헌 범위 + PT 검수(8은 절대 고정 아님, 10까지 허용) |

**핵심 인용 프레이밍(포트폴리오용):** "임상 문헌(Ryu 1991 등)의 기능적 ROM과 증상 유도 재활 원칙에 기반한 적응형 루틴."

## 2. 신호등 프레임워크 (문헌 → 앱 매핑)

문헌의 traffic-light를 우리 앱 데이터(conditions·comp·측정)에 매핑:

| 신호 | 문헌 정의 | 우리 앱 신호 | 조치 |
|---|---|---|---|
| 🟢 초록 | 통증 0-2, 다음날 같거나 나음 | condition=good, 다음날 악화 없음, comp 낮음 | 진행 가능(조건 충족 시 상승) |
| 🟡 노랑 | 통증 3-4, 다음날 가라앉음, 자세 흐트러짐 | condition=soso, comp 잦음 | 현 강도 유지 |
| 🔴 빨강 | 통증 5+, 날카로움, 보상동작, 다음날 악화 | condition=stiff, 또는 다음날 악화, comp 매우 높음 | 순한 코스, 부하 감소 |

**우리 앱은 통증 수치(0-10)를 직접 안 받음** → condition(good/soso/stiff)과 comp(보상동작 비율), 측정 추이로 대리.

## 3. 데이터 (이미 존재 — 새로 안 만듦)

전부 store에 이미 쌓이는 중. 맞춤은 이걸 **읽기만** 함:
- `measurements`: [{at, hand, flex, ext, rom}] — 방향별 각도. "약함" 판정 소스
- `conditions`: [{at, condition}] — 신호등 소스
- `routineLog`: [{at, done, total}] — 세션 견딤 카운트 소스
- 보상동작 비율(comp / lastCompRatio) — 신호등 보조

**추가할 상태**(store, 최소):
```
adapt: {
  focus: 'flex'|'ext'|null,   // 현재 우선 강화 방향
  doseLevel: { [guideId]: n }, // 운동별 현재 강도 단계(0=기본)
  toleratedStreak: 0,          // 연속 잘 견딘 세션 수
  lastImproveShownAt: null,    // 긍정 신호 마지막 표시일(도배 방지)
}
```

## 4. 규칙 세트 (구현 사양)

### 4.1 "약함" 판정 → focus 방향
- 최근 측정에서 **flex < 40° → focus='flex'**, **ext < 40° → focus='ext'**
- 둘 다 미만이면 더 낮은 쪽
- **둘 다 40°↑이면** → "부족" 아님. 낮은 쪽을 focus로 두되 **작은 차이만**(운동 1개 reps +1 정도), 긍정 톤 유지
- 측정 없으면 focus=null → 기본 코스 그대로

### 4.2 방향 특이적 조정 (빼지 말고 반복 늘리기)
- 6개 코스는 **유지**(특정 운동 제거 금지 — 안전). focus 방향 운동의 reps만 소폭↑
- flex focus → flex_ext의 reps +1~2 / ext focus → 동일 운동에서 신전 비중 강조
  (flex_ext는 굴곡·신전 복합이므로 reps로 총량 조절)

### 4.3 진행 (보수적 · 증상 기반)
- **기본: 현 강도 유지.** 아래 **3조건 모두** 충족 시에만 1단계 상승:
  - 최근 **3세션 연속** 무난 수행 (condition≠stiff)
  - **다음날 증상 악화 없음** (어제 대비 condition 하락 없음)
  - **보상동작 없음 또는 적음** (comp 낮음)
- 상승 = **reps +1 또는 hold +2~5초 중 하나만**(동시 금지)
- **상한(cap) — 단계적**(물리치료사 검수 반영):
  - 기본 8회 전후에서 시작
  - 잘 견디면 **10회까지** 상승 허용
  - 그 이상은 증상 반응 보고 결정(자동 상승 상한 = 10)
  - hold는 10~15초면 충분 (상한 15초)
  - ※ 8회 절대 고정 아님 — 손목 웰니스 성격상 8~10 범위는 무리 없음

### 4.4 후퇴 (안전)
- 다음날 condition 악화(예: 어제 good → 오늘 stiff) → **1단계 하강**, 안정될 때까지 유지
- condition=stiff이면 → 순한 코스(기존 gentleCourse) + 해당 방향 reps 원위치

### 4.5 긍정 신호 (개선 시에만, 가끔)
- 측정 추이가 상승(직전 대비 flex/ext/rom 개선) + 최근 잘 견딤 → **"손목이 부드러워지고 있어요 ✨"** 류 1회
- **도배 방지**: lastImproveShownAt로 최소 간격(예: 3일) 두기
- 악화·정체는 **표시 안 함**(부정 프레이밍 금지)

## 5. 기존 코드에 얹는 방법

`routine.js`의 `courseIds`·`getTodayRoutine`이 진입점. 지금 `isGentleDay`(어제 stiff면 순한 코스) 로직이 있음 → 이를 확장:

- `computeFocus(state)` 신규: measurements 읽어 focus 방향 산출 (§4.1)
- `computeDose(state, guideId)` 신규: adapt.doseLevel + 진행/후퇴 규칙으로 reps/hold 반환 (§4.3~4.4)
- `getTodayRoutine`에서: 코스 구성 시 focus 방향 운동에 dose 반영, gentle 판정에 측정 기반 red 신호 추가
- 클립 재생(stepEngine)에서 reps/hold를 config 고정값 대신 computeDose 결과로

**측정·인식 엔진(measurement.js·tracking.js·stepEngine 판정 로직)은 불변.** 값을 읽어 루틴 파라미터만 조정.

## 6. 구현 단계 (단계별 커밋 · 안전 순)

1. **adapt 상태 추가** (store, 화면 변화 0) — focus/doseLevel/toleratedStreak/lastImproveShownAt
2. **약함 판정 + focus** (§4.1) — computeFocus, 아직 루틴에 반영만(reps 소폭)
3. **순한 코스 측정 기반 확장** (§4.4 red) — 기존 isGentleDay에 red 신호 추가 (제일 안전, 기존 로직 확장)
4. **방향 특이적 reps 조정** (§4.2) — focus 방향 reps 소폭↑, 상한 적용
5. **진행/후퇴 규칙** (§4.3~4.4) — 3세션 견딤 → 1변수 상승, 다음날 악화 → 하강
6. **긍정 신호** (§4.5) — 개선 시에만, 간격 두고

각 단계 독립 검증. 3번(순한 코스 확장)이 기존 로직 확장이라 제일 쉬움 → 여기부터 시작 추천.

## 7. 검증 관점

- **안전**: 어떤 경우에도 reps/hold가 상한 초과 안 함. stiff 다음날 반드시 하강. 측정 없으면 기본 코스.
- **조용함**: 사용자 화면에 "약함/부족/N%" 노출 0. 긍정 신호만 가끔.
- **엣지**: 측정 0회(신규), 측정 1회(추이 없음), 손 좌우 다름(hand별 분리 필요할 수 있음 — v2), 오래된 측정(예: 30일 전 값은 신뢰도↓)
- **되돌리기**: adapt 로직 꺼도 기본 코스로 정상 동작(안전 폴백)

## 8. 포트폴리오 메모

- "물리치료사 배경 + 임상 문헌 근거"로 설계한 적응형 알고리즘 = 강력한 차별점
- 근거 인용(Ryu 1991, 증상 유도 진행, 24h rule) 명시 → "감이 아니라 근거"
- "조용한 맞춤 + 긍정 신호만" = 웰니스 UX 철학(불안 안 주기)의 구현 사례
- 관심사 분리: 인식 엔진 불변 + 루틴 레이어만 적응 = 설계 역량 증거
