# 보이는 돌봄 설계 — "오늘의 별자리 한마디"

손목 웰니스 앱 "오늘의 별자리" · UX 설계 · 2026-07
목적: **안 보이던 적응형 맞춤을 "앱이 날 챙긴다"는 체감으로** 전환. 심심함·식상함도 함께 해소.

---

## 0. 지배 원칙

**"돌봄은 보이게, 심판은 숨기게."**
모든 내부 적응 신호를 **긍정 또는 따뜻한-중립**으로 번역해 노출한다.
"조용한 맞춤"은 *부정 프레이밍 금지*이지 *투명인간 되기*가 아니다 — 지금은 극단적으로 안 보여서
정교한 엔지니어링이 체감 가치에 기여를 못 한다. 그 사이의 "긍정으로 느껴지게"를 채운다.

**절대 노출 금지(가드레일):** 약함/부족, 수치 성적표, 정상 대비 %, 남과 비교, 실패·놓침 강조,
하락 직접 노출("나빠졌어요"), 재촉.

---

## 1. 핵심 장치 — "오늘의 별자리 한마디"

하루 한 번, 홈에서 **오늘의 별자리가 당신 상태에 맞춘 따뜻한 한 줄**을 건넨다.
이 한 줄이 내부 적응 신호(focus·dose·gentle·개선·꾸준함)를 긍정으로 번역한다.

- 앱 이름 "오늘의 별자리"의 값을 다 쓴다(별자리 = 수집 그릇 → 말 거는 존재).
- 안 보이던 적응형이 "앱이 날 본다"로 드러난다.
- 차분함 유지: **하루 한 줄, 홈에서만.**

### 1.1 우선순위 (딱 하나만 선택)

| 순위 | 조건(내부 신호) | 톤 |
|---|---|---|
| 1 | 개선 감지(improveSignal, 간격 충족) | 축하 |
| 2 | 순한 날(gentleReason: stiff / red) | 다독임 |
| 3 | 레벨업(lastDoseAction = up) | 성장 |
| 4 | 약한 방향(focus 있음) | 오늘의 포커스(긍정) |
| 5 | 꾸준함(toleratedStreak/스트릭 높음) | 인정 |
| 6 | 기본(중립) | 별자리 flavor |

같은 날 = 같은 문구(날짜 기반 안정 선택), 단계별 문구 2~3개 로테이션.

### 1.2 문구 시작 세트 (나중에 다듬기)

**1) 개선**
- "손목이 조금씩 부드러워지고 있어요. 오늘의 별도 함께 반짝여요 ✨"
- "지난주보다 한결 편해졌네요. 잘 하고 있어요 🌟"

**2) 순한 날 — stiff(자기보고)**
- "어제 좀 뻐근했죠? 오늘은 별도 천천히 흐르게 둬요 🌙"
- "무리하지 않아도 괜찮아요. 오늘은 가볍게 🌙"

**2') 순한 날 — red(측정 추론, 중립)**
- "오늘은 살살 가볼까요, 무리 없이 🌙"
- "천천히도 충분해요. 오늘의 별자리와 함께 🌙"

**3) 레벨업**
- "손목이 한 걸음 더 갈 준비가 됐네요. 오늘 살짝 더 해볼까요?"
- "부쩍 편해졌어요. 오늘은 조금 더 담아봐요 💫"

**4) 오늘의 포커스(focus → 운동명, 부족이 아니라 방향으로)**
- "오늘은 '굽힘'에 살짝 더 마음 써봐요"
- "오늘의 포커스는 '{운동}'이에요. 천천히 함께 해요 🎯"

**5) 꾸준함**
- "요즘 참 꾸준하네요. 오늘의 별자리도 당신을 기다렸어요"
- "{n}일째 함께하고 있어요. 오늘도 반가워요 🐾"

**6) 기본(중립)**
- "오늘의 별자리는 '{이름}'이에요. 잔잔한 하루 보내요"
- "오늘도 만나서 반가워요. 손목, 잠깐 챙겨볼까요? 🐾"

## 2. 곁들이는 "보이는 돌봄" 3가지 (이벤트성 · 단계 분리)

도배 금지 — 사건이 있을 때만, 기존 긍정 신호처럼 간격 두고.

**① 완성 순간 payoff (심심함이 죽는 자리)**
별자리 완성 시: 별들이 **선으로 이어지며 그림 완성** + 은은한 반짝임 + **고양이가 하늘을
올려다봄** + "오늘의 '{이름}' 완성 🌟". 차분하되 payoff는 있게. *(시각 작업 — 별도 단계)*

**② 측정 결과 = 선물 프레이밍**
개선 시 "지난주보다 굽힘이 조금 늘었어요" + **별 하나 선물처럼 추가**. 하락은 노출 안 함
(추세선만). 측정이 "성적표"가 아니라 "응원"이 된다.

**③ 루틴 '오늘의 포커스' 태그**
focus 방향 운동에 작은 🎯 "오늘의 포커스" 뱃지. "너 약해"(✗) → "오늘은 여기 신경 써봐요"(✓).

## 3. 절제 규칙

- **코어는 "하루 한 줄(별자리 한마디)" 하나.** 나머지(payoff·선물·태그)는 사건성.
- 화면마다 메시지 도배 금지 — 차분함이 최대 차별점.
- 목표: "시끄럽게"가 아니라 **"몇 군데 잘 놓인 따뜻함."**

---

## 4. 구현 매핑

- **한마디 선택 = 순수 함수** `dailyStarMessage(state, date)` — 상태를 읽어 우선순위대로 한 줄
  반환. 저장·부수효과 없음. 문구 세트는 한 곳(config/상수 모듈)에 모아 수정 쉽게.
- 상태 소스: `getAdapt`(focus·focusSoft·lastDoseAction·toleratedStreak), `gentleReason`,
  `improveSignal`(개선 tier로 편입·재사용 → 중복 메시지 방지), `sky`(오늘 별자리 이름),
  `currentStreak`. focus→운동명은 guideData name 매핑.
- 표시: 홈 렌더(기존 speech/improveSignal 자리). 같은 날 재렌더 = 같은 문구(깜빡임 없음).

## 5. 단계

1. **한마디 시스템**(순수 함수 + 홈 표시 + 문구 세트) ← 첫 구현
2. 완성 payoff 애니(시각 — 브라우저 확인 필요)
3. 측정 선물 프레이밍 + 오늘의 포커스 태그
