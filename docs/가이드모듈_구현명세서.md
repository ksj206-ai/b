# 숏폼 운동 가이드 모듈 — 구현 명세서 (Claude Code 인수인계용)

> 손목 웰니스 앱(wrist-garden.html)에 추가할 **캔버스 애니메이션 기반 숏폼 운동 가이드 모듈**의 기획·구현 명세.
> 이 문서만 보고 구현 가능하도록 작성됨. 기존 코드의 인식 로직을 최대한 재사용한다.

---

## 0. 배경 (프로젝트 컨텍스트)

- **앱 정체성 (재정립됨):** 게임이 메인이 아니라, ① 숏폼 운동 가이드(메인) ② ROM 측정·기록 ③ 미니게임(서브) 3축의 **데일리 손목 웰니스 루틴 앱**
- **타겟:** 컴퓨터·스마트폰을 많이 쓰는 직장인·학생 (건강한 일반인의 피로 완화)
- **이 모듈의 목표:** 일러스트 손이 운동 시범을 보여주고, 사용자가 따라하면 웹캠 인식으로 회수를 체크해주는 **1~3분 숏폼 가이드**
- **저작권:** 외부 영상 사용 금지. 전부 캔버스 코드로 자체 제작 (기존 앱 그림체 유지)
- **의료법 주의:** "치료·진단·처방" 표현 금지. "피로 완화·스트레칭·가이드" 톤. 각 가이드에 "통증 시 즉시 중단" 문구. 인식 결과는 판정이 아닌 참고.

## 1. 화면 구성

```
┌─────────────────────────────────┐
│  ✋ 손목 굽힘·폄 스트레칭  (1/3)  │  ← 가이드명 + 스텝 진행
│                                 │
│   [시범 손 애니메이션 영역]       │  ← 캔버스, 무한 반복 재생
│                                 │
│   "천천히 아래로 굽혔다,          │  ← 스텝 안내 텍스트
│    위로 펴세요 (5회)"            │
│   ⚠ 통증이 느껴지면 즉시 중단     │
│                                 │
│   내 동작: ●●●○○  (3/5회)       │  ← 웹캠 인식 카운트
│   [건너뛰기]        [다음 →]     │
└─────────────────────────────────┘
```

- 시범 손은 사용자와 무관하게 자기 페이스로 **무한 반복**
- 사용자 카운트는 사용자 페이스로 독립 인식 (박자 강요 금지 — 게임이 아니라 가이드)
- 진입점: 홈 화면에 "가이드" 모드 버튼 (게임 모드와 나란히)

## 2. 가이드 손 모델 — "파라미터로 조종하는 인형"

손을 이미지 여러 장이 아니라 **파라미터 몇 개로 조종되는 하나의 벡터 손**으로 구현한다.
모든 운동 = 파라미터 값의 시간 변화.

### 조종 파라미터
| 파라미터 | 범위 | 용도 |
|---|---|---|
| `wristAngle` | -45(굽힘) ~ +45(폄) | 굽힘·폄 |
| `devAngle` | -30(요측) ~ +30(척측) | 좌우 편위 |
| `curl[5]` | 0(쫙) ~ 1(주먹), 손가락별 | 활주·악력 |
| `curlTip` | 0 ~ 1 (끝마디만 구부림) | 갈고리 표현 |
| `spread` | 0(모음) ~ 1(쫙 벌림) | 손가락 벌리기 |
| `pinchGap` | 0(붙음) ~ 1(벌림) | 핀치 |

### 뷰(시점) — 2종만
- **`side` (옆모습, 팔뚝+손날):** 굽힘·폄용. 각도 변화가 가장 잘 보임
- **`front` (정면 손바닥):** 편위·손가락·핀치용

**설계 원칙:** 시범 손의 시점 = 사용자가 카메라에 취해야 할 자세와 동일하게. (시범은 옆인데 사용자는 정면 → 인식 불일치 방지)

### 그리기 기본 구조 (side 뷰 예시)
```javascript
function drawGuideHand(ctx, p, view){ // p = 파라미터 객체
  if(view==="side"){
    const wx=..., wy=...;                        // 손목 기준점
    // 팔뚝: 고정 막대 (둥근 캡)
    // 손: ctx.translate(wx,wy); ctx.rotate(p.wristAngle*Math.PI/180);
    //     손바닥 막대 + 손가락(살짝 curl) + 엄지
  } else { /* front: 손바닥 원 + 손가락 5개 선분(관절 2개), curl/spread/pinchGap 반영 */ }
}
```
- 손가락은 선분 2~3개(마디) 연결로 충분. `curl`은 각 마디의 굽힘 각도로 매핑
- 처음엔 단순 막대 손으로 기능 완성 → 이후 디테일(손톱·소매 등) 추가하는 순서
- **주의(기존 앱 이슈):** 이 환경은 이모지를 fillStyle 색으로 렌더할 수 있음 → 이모지 대신 벡터 드로잉 권장. 이모지 쓸 경우 fillText 직전 불투명 fillStyle 지정 필수

## 3. 애니메이션 시스템 — 매끄러움의 공식

**매끄러움 = ① 절대시간 기반 + ② 이징(가감속) + ③ 홀드(정지) 구간**

### ① 절대시간 기반 (필수)
```javascript
const progress = ((now - animStart) % durationMs) / durationMs; // 0~1 반복
```
- dt 누적 금지 (프레임 드랍 시 끊김 — 기존 앱에서 검증된 원칙)

### ② 키프레임 + 이징
```javascript
const easeInOut = t => t<0.5 ? 4*t*t*t : 1 - Math.pow(-2*t+2,3)/2;
// 키프레임: [시간(s), 파라미터부분집합] 배열
// 재생: 현재 시간이 속한 구간을 찾아, 구간 내 진행률에 easeInOut 적용해 파라미터 보간
```

### ③ 홀드 구간 (가장 중요)
- 끝범위에서 **0.8~2초 정지**하는 키프레임을 반드시 포함
- 이게 없으면 "휘적휘적", 있으면 "스트레칭 시범"이 됨 (임상적으로도 끝범위 유지가 핵심)

## 4. 운동별 키프레임 명세

### ① 손목 굽힘·폄 (view: side)
```
[0s, {wristAngle:0}] → [1.5s, {wristAngle:-35}] → [2.5s, {wristAngle:-35}]  // 굽힘+홀드1s
→ [4s, {wristAngle:+30}] → [5s, {wristAngle:+30}] → [6s, {wristAngle:0}]   // 폄+홀드1s
```
- 손가락은 curl=0.15 고정 (완전 뻣뻣하면 로봇 같음)

### ② 좌우 편위 (view: front)
```
[0,{devAngle:0}] → [1.2,{devAngle:-20}] → [2.0,{devAngle:-20}]
→ [3.5,{devAngle:+25}] → [4.3,{devAngle:+25}] → [5.3,{devAngle:0}]
```
- 요측 20° / 척측 25° 비대칭 (실제 인체 가동범위 비율 반영)

### ③ 힘줄 활주: 쫙→갈고리→주먹 (view: front)
- 3자세 파라미터:
  - 쫙: `{curl:0, curlTip:0, spread:0.8}`
  - 갈고리: `{curl:0.15, curlTip:1.0}`
  - 주먹: `{curl:1.0, curlTip:1.0}`
- 각 자세: 0.8s 전환 → **1.2s 유지** → 0.4s 중립 경유 → 다음 자세
- **디테일:** 손가락 5개를 새끼→검지 순 0.05s 시차로 움직이면 자연스러움

### ④ 핀치 (view: front)
```
[0,{pinchGap:1}] → [0.8,{pinchGap:0}] → [2.8,{pinchGap:0}] → [3.4,{pinchGap:1}]
```
- 붙는 순간 접점에 작은 반짝 1개 (힘주는 중 표시)

### ⑤ 악력 유지 (view: front)
```
[0,{curl:0.2}] → [0.6,{curl:1.0}] → 홀드 3s 동안 curl을 1.0±0.03 미세 진동 → [풀기]
```
- 미세 떨림이 "힘주고 있음"을 표현하는 핵심 트릭

### ⑥ 손가락 벌리기 (view: front)
```
[0,{spread:0.3}] → [1,{spread:1.0}] → [2.5,{spread:1.0}] → [3.5,{spread:0.3}]
```
- 최대 벌림 시 손가락 사이 옅은 빛살 (햇빛 컨셉 연결, 선택)

## 5. 스텝 엔진 & 데이터 구조

```javascript
const GUIDES = [{
  id:"flex_ext", name:"손목 굽힘·폄 스트레칭", view:"side",
  steps:[
    {type:"intro",  text:"팔을 편하게 두고 손에 힘을 빼요", dur:3, pose:{wristAngle:0}},
    {type:"follow", text:"천천히 굽혔다 펴세요", reps:5, detect:"flexExt",
     anim:[[0,{wristAngle:0}],[1.5,{wristAngle:-35}],[2.5,{wristAngle:-35}],
           [4,{wristAngle:30}],[5,{wristAngle:30}],[6,{wristAngle:0}]]},
    {type:"outro",  text:"잘했어요! 가볍게 털어주세요", dur:3, pose:{wristAngle:0}}
  ]},
  // tendon_glide, deviation, pinch_hold, grip_hold, finger_spread ...
];
```

- 스텝 type: `intro`(자동 진행, dur초) / `follow`(인식 카운트로 진행, 건너뛰기 가능) / `outro`(자동)
- 구현 모듈 3분리:
  1. **drawGuideHand(params, view)** — 파라미터로 손 그리기
  2. **animPlayer** — 키프레임 보간 + 이징 재생
  3. **stepEngine** — 스텝 진행·텍스트·인식 카운트·완료 처리
- 가이드 추가 = GUIDES 데이터만 추가 (플레이어는 공용)

## 6. 따라하기 인식 — 기존 코드 재사용

기존 앱의 인식 변수·판정 로직을 그대로 사용한다 (새 개발 최소화):

| detect 키 | 재사용 로직 | 비고 |
|---|---|---|
| `flexExt` | `rel` + armed(중립 복귀) 게이트 (두더지 게임 방식) | 굽힘 rel<0 / 폄 rel>0. **폄 목표는 낮게** (2D 한계, 기존 상한 14° 참고) |
| `deviation` | `rel` (편위 모드) 좌우 왕복 | 물주기 게임 방식 |
| `tendonGlide` | `grip`, `tipMCP` 3자세 판정 | 리듬게임 판정 재사용 |
| `pinchHold` | `pinch<0.34` + 유지 타이머 | 핀치 게임(2초 유지) 재사용 |
| `gripHold` | `grip<1.2` + 중립 + 유지 | 영양제 게임 재사용 |

**UX 원칙:**
1. 시범과 카운트는 **독립** (박자 강요 금지)
2. 목표 각도는 게임보다 **관대하게** (가이드는 성공 경험 우선)
3. 피드백은 **조용하게** — 회차 점(●●●○○) 채움 + 은은한 체크음. 게임식 파티클 금지 (모드 톤 구분)
4. **인식 실패 탈출구 필수:** follow 스텝에서 15초간 인식 0회면 안내 표시 → [계속 시도] / [손동작 없이 진행]. 인식이 가이드를 막으면 안 됨
5. 보상동작(comp) 경고는 표시하되 카운트를 막지 않음 (부드럽게 안내만)

## 7. 마감 디테일 (우선순위 순, 선택 구현)

1. **고스트 목표:** 시범 손의 끝위치에 반투명 실루엣 미리 표시 (어디까지 가는지 예측)
2. **호흡 텍스트:** 홀드 구간에 "후— 숨 내쉬며 유지" (웰니스 톤)
3. **스텝 전환 모핑:** 운동 전환 시 0.5s간 중립 자세로 파라미터 보간 (뿅 바뀜 방지)
4. **모션 트레일:** 손끝 궤적 0.3s 잔상 (움직임 방향 가시화)
5. 시범 속도 조절 (느리게/보통)

## 8. 구현 순서 (권장)

1. `drawGuideHand` side 뷰 (팔뚝+손, wristAngle만) — 가장 단순
2. animPlayer (키프레임+이징) → 굽힘·폄 시범 무한 반복 확인
3. stepEngine (intro/follow/outro, 텍스트, 진행)
4. flexExt 인식 연결 (rel 재사용) → **가이드 1개 완성이 마일스톤**
5. front 뷰 손 + 나머지 운동 데이터·애니메이션 추가
6. 홈 화면 "가이드" 진입 버튼 + 완료 기록 저장 (추이 리포트와 연결 예정)

**난이도 참고:** 1~2 쉬움 / 3 중간(새 구조) / 4 쉬움(재사용) / 5부터는 데이터 추가 반복

## 9. 기존 코드 참고 위치 (wrist-garden.html)

- 인식 변수: `rel`(손목 각도), `grip`/`tipMCP`/`spread`/`pinch`(손가락), `comp`(보상동작)
- 판정 재사용처: `s.mode==="dig"`(flexExt+armed), `"glide"`(3자세), `"pinch"`(핀치 유지), `"isopump"`(악력)
- 원칙: 절대시간 애니메이션, 편집 후 문법 검사(node --check), 이모지 투명 이슈 주의
- 좌표계: W=960, H=540, groundY=348
