## 1. 문제 정의

정면 촬영(손바닥이 카메라를 향한 자세)에서 손목을 굽히면 움직임이 카메라 축(z) 방향이라 2D 좌표만으로는 각도가 심하게 과소평가된다(원근 단축). 단, 정보는 사라지지 않고 두 곳에 남는다 — 화면상 길이가 `L₀cosθ`로 줄고, 깊이 `Δz = L₀sinθ`가 생긴다. 측정 알고리즘은 이를 역산한다.

한편 **요측/척측 편위는 정면 자세에서 화면 평면 안의 움직임**이므로 원근 단축이 없고 2D 벡터 각도만으로 정확히 측정된다. 즉 같은 자세·같은 캘리브레이션으로 4개 항목(굴곡·신전·요측·척측)을 모두 측정한다.

## 2. 측정 방식 결정

### 2.1 채택: 방법 B (길이 비율 역산) + z 부호 하이브리드

- **각도 크기**: `θ = acos(L / L₀′)` — 가장 정확한 x, y 좌표만 사용
- **굴곡/신전 방향**: 중지 MCP와 손목의 z 비교(부호 판별에만 사용, 노이즈에 강함)
- **보조 검증**: 방법 C(벡터 사영)를 백그라운드 병산, B와 크게 어긋나면 "신뢰도 낮음" 표시

### 2.2 기각 및 보류

| 항목 | 판단 | 사유 |
|---|---|---|
| 방법 A (3D 벡터 내적) | 기각 | z 노이즈 직행(±10° 널뜀), z 스케일 계통 오차, Hands 모델에 전완 벡터 없음 |
| 방법 C 단독 사용 | 기각 | Δz 스케일 불확실로 절대 각도 신뢰 불가. 검증 보조로만 |
| Pose 모델 병행(각도 계산) | 기각 | 정면에선 전완도 똑같이 단축, Hands/Pose z 좌표계 불일치, 연산 부담 |
| Pose 모델(품질 관리) | **v2 보류** | 보상동작 감지·전완 고정 검증. 임상적 차별점이나 MVP 범위 밖 |
| 굴곡/신전을 2D 벡터 회전(Δθ = θ−θ₀)으로 계산 | 기각 (참고문서 3회 반복 등장) | 측면 촬영 전용 수식. 정면에선 원근 단축으로 무력화. 추후 "측면 촬영 모드" 추가 시 재활용 가능. 각 문서의 임계값·판정 구조만 발라내어 사용 |
| 중지 TIP(12번)으로 손목 각도 측정 | 기각 | 손가락 굽힘이 손목 각도로 오염. MCP(9번) 고정 |

### 2.3 방법 B의 수학적 특성 (설계 근거)

- **강한 구간**: 30~80° — 길이 변화가 커서 안정. 핵심 ROM 구간과 일치
- **약한 구간**: 0~10° — 길이 변화 1.5% 미만이라 노이즈가 각도로 증폭 → 데드존 처리. 90° 근처 — 손이 점으로 보여 추적 불안정 → 신뢰도 게이트 처리

## 4. 캘리브레이션 설계

### 4.1 개념

사용자별 기준값 영점 조정(goniometer 0점 맞추기와 동일). **"완벽한 기준 의식"이 아니라 "대략적 출발값 절차"** — 정확도는 3차 방어(자동 보정)가 세션 중 수렴시킨다.

### 4.2 저장값

| 값 | 정의 | 용도 |
|---|---|---|
| L₀ | 중립 손등 길이 dist2D(0,9) | 굴곡/신전 역산 기준 |
| W₀ | 중립 손바닥 폭 dist2D(5,17) | 거리 정규화 기준 |
| θ₀ | 중립 손 방향 atan2(y₉−y₀, x₉−x₀) | 편위 측정 기준 |
| D₀ (v2 추가) | 각 손가락 TIP-MCP 거리 4개 | 손가락 동작(펴기/주먹) 판정 기준 |

### 4.3 절차 (1차 방어: UX + 안정 판정)

1. 반투명 손 실루엣 가이드(SVG 오버레이, opacity 0.3, 화면 40~50%) 표시
2. 안내 문구: **"팔꿈치를 책상에 올린 편한 자세에서, 아프지 않은 범위에서 손바닥을 최대한 펴서 카메라로 향해주세요"** — 측정 자세와 동일한 자세 유도 + 통증 강요 금지
3. 최근 30프레임의 L 표준편차가 평균의 2~4% 이내면 안정 판정
4. 안정 통과 시 30프레임 **중앙값**을 저장(튄 프레임 오염 방지)
5. 실루엣 색 피드백: 회색(미감지) → 주황(조건 미달+안내) → 초록(3-2-1 카운트다운)

실루엣은 위치 유도용일 뿐 판정 기준이 아님. 판정은 랜드마크로만.

### 4.4 2차 방어: 기하학 검사

- **z 평탄도**: 중지 MCP와 손목의 z 차이가 임계값 초과 → "손목을 펴주세요", 저장 거부
- **인체 비율**: L₀/W₀가 정상 범위 이탈 → 재시도 요청 (범위는 실측으로 튜닝)

### 4.5 3차 방어: 세션 중 자동 보정 (핵심)

L₀는 정의상 세션 최대 정규화 길이. 더 큰 값이 **5프레임 이상 유지**되면 갱신:

```javascript
const normL = d2(lm[0], lm[9]) / (W / this.W0);
if (normL > this.L0) { if (++this.holdCount >= 5) this.L0 = normL; }
else this.holdCount = 0;
```

캘리브레이션이 부정확해도 운동 중 손목을 편 순간이 지나가면 기준이 스스로 교정됨.

### 4.6 문제 대응 결정사항

| 문제 | 대응 |
|---|---|
| 안정 판정 영영 미통과 → 이탈 | 조건 관대하게 시작(4%) + **10초 미통과 시 "이대로 시작하기" 탈출구 버튼** |
| 실루엣 맞추려다 자세 붕괴 | 자세 중심 안내 문구, 실루엣은 느슨한 참고용 |
| 왼손 사용자 | handedness 감지 → 실루엣 CSS `scaleX(-1)`, 또는 시작 시 손 선택. (v2.1) 선택/감지된 손은 세션 전체의 hand 태그로 저장돼 기록·기준값이 좌우 분리됨. L₀/W₀ 비율도 손별로 각각 저장(좌우 비례가 다를 수 있음) |
| 재활 사용자는 완전 중립 불가 | 버그 아닌 기능: 개인 최대치가 0점, 변화 추세 측정. 자동 보정이 회복 따라 기준 상향 |
| 매 세션 캘리브레이션 마찰 | **L₀/W₀ 비율(손 고유 비례, 거리 무관)을 localStorage 저장** → 2회차부터 현재 W만 재서 즉시 복원. 풀 캘리브레이션은 최초 1회 + 이상 시만 |
| 거울 모드 좌표 불일치 | 랜드마크 오버레이 시 `x = 1 − x` 변환 필수 |
| 측정값 지속 이상 | 측정 화면 구석 재캘리브레이션 버튼 상시 배치 |

구현 순서: 1차 → 3차 → 2차(임계값 튜닝 필요, 마지막)

## 5. 측정 파이프라인 (매 프레임)

### 5.1 굴곡/신전

```
0. 캘리브레이션(세션당 1회) → L₀, W₀, θ₀, D₀
1. 거리 정규화:  L₀′ = L₀ × (W / W₀)
2. 각도 계산:    θ = acos( min(1, L / L₀′) )    ← 클램프 필수(NaN 방지)
3. 부호: z차이 EMA 후 판별 (MCP가 가까우면 굴곡+, 멀면 신전−)
   데드존: |θ| < 10° → 0   /   스무딩: EMA 계수 0.8
4. 신뢰도 게이트(걸리면 프레임 기록 제외):
   손바닥 폭 1프레임 5% 급변(회전 의심) / handedness score < 0.8 / ratio 비정상
5. 세션 기록: 유효 프레임 상위 5% 평균 = 그날의 최대 ROM (단일 최댓값 금지)
```

**(v2) 반환값에 rawAngle 추가**: `{angle, rawAngle, valid}` — angle은 데드존 적용값(일반 측정용), rawAngle은 데드존 미적용값(중립 유지 게임 등 0° 근처 정밀도가 필요한 기능용). 데드존과 중립 게임(±5°)의 충돌을 이 이중 반환으로 해소.

### 5.2 요측/척측 편위

```
θ_now = atan2(y₉ − y₀, x₉ − x₀)
θ_dev = θ_now − θ₀   (−180~180° wrap 정규화)
부호: 엄지 방향 = 요측, 소지 방향 = 척측 (handedness에 따라 반전)
```

- 참고 문서들의 arcsin(Δx/d) 비례 모델은 atan2 차이로 대체(근사·정규화 우수)
- EMA·게이트·세션 기록은 굴곡/신전 파이프라인 공유
- L/L₀′ 비율이 크게 떨어지면 "손목을 굽히지 말고 좌우로만" 안내(복합 동작 배제)

### 5.3 (v2) 편위 자세 통일 결정 — 중요

참고 문서들(임상 관행)은 편위 운동을 "손바닥을 아래로 책상에 올리고" 안내하지만, 이 자세에서는 편위가 원근 단축을 맞아 측정이 무너진다. **결정: 측정이 있는 모든 동작은 손바닥-카메라 자세로 통일.** 편위는 어느 평면에서든 같은 관절 운동이라 임상적으로 무방. 가이드 애니메이션 키프레임도 반드시 이 자세로 제작. "손바닥 아래" 자세는 측정 없는 순수 스트레칭 클립에서만 사용.

### 5.4 손가락 동작 판정 (준비운동용, v2 추가)

거리 비율 기반이라 정면 자세에서도 안정적으로 작동(방법 B와 같은 원리 계열):

- 손 펴기: 평균 TIP-MCP 거리 / D₀ ≥ 1.3
- 주먹: TIP-손목 거리 비율 ≤ 0.5
- 판정은 §6의 hold 방식(0.5초 유지) 적용

### 5.5 핵심 코드 골격

```javascript
// js/rom/measure.js
const d2 = (a, b) => Math.hypot(a.x - b.x, a.y - b.y);

export class RomMeasure {
  constructor(L0, W0, theta0) {
    this.L0 = L0; this.W0 = W0; this.theta0 = theta0;
    this.smoothed = 0; this.zDiff = 0;
    this.prevW = W0; this.holdCount = 0; this.records = [];
  }

  update(lm, score, mode /* 'flex' | 'dev' */) {
    const W = d2(lm[5], lm[17]);
    if (score < 0.8 || Math.abs(W - this.prevW) / this.prevW > 0.05) {
      this.prevW = W;
      return { angle: this.smoothed, rawAngle: this.smoothed, valid: false };
    }
    this.prevW = W;

    const L = d2(lm[0], lm[9]);
    const L0p = this.L0 * (W / this.W0);

    // 자동 보정 (3차 방어)
    const normL = L / (W / this.W0);
    if (normL > this.L0) { if (++this.holdCount >= 5) this.L0 = normL; }
    else this.holdCount = 0;

    let theta;
    if (mode === 'flex') {
      theta = Math.acos(Math.min(1, L / L0p)) * 180 / Math.PI;
      this.zDiff = this.zDiff * 0.8 + (lm[0].z - lm[9].z) * 0.2;
      theta *= this.zDiff > 0 ? 1 : -1;
    } else {
      const now = Math.atan2(lm[9].y - lm[0].y, lm[9].x - lm[0].x);
      theta = ((now - this.theta0) * 180 / Math.PI + 540) % 360 - 180;
    }

    const raw = this.smoothed * 0.8 + theta * 0.2;        // EMA
    this.smoothed = raw;
    const deadzoned = Math.abs(raw) < 10 ? 0 : raw;        // 데드존은 표시용
    this.records.push(deadzoned);
    return { angle: deadzoned, rawAngle: raw, valid: true };
  }

  sessionMax() {  // 상위 5% 평균
    const s = [...this.records].sort((a, b) => b - a);
    const top = s.slice(0, Math.max(1, Math.floor(s.length * 0.05)));
    return top.reduce((a, b) => a + b, 0) / top.length;
  }
}
```

## 6. 판정 계층 — judge.js (v2 신설)

측정("몇 도인가")과 판정("운동을 성공했는가")을 분리한다. measure.js의 `{angle, rawAngle, valid}` 스트림을 입력받는 상태 머신.

### 6.1 판정 패턴 두 가지

**패턴 A — Hold time (재활 동작용)**: "임계각 이상 + 유효 프레임"이 N초 연속이면 성공.
- valid=false 프레임 발생 시 타이머 **리셋이 아니라 일시정지** (짧은 추적 끊김으로 성공이 날아가는 억울함 방지)
- 기본 유지 시간: ROM 동작 1초, 스트레칭 15초, 손가락 동작 0.5초

**패턴 B — 프레임 비율 (게임용)**: 목표 각도 대비 오차 ≤5°인 프레임을 "정답 프레임"으로 정의, **1초간 정답 프레임 비율 ≥ 80%면 성공.** hold보다 노이즈에 관대해 게임 판정에 적합.

### 6.2 3단계 등급

| 등급 | 굴곡/신전 기준 | 의미 |
|---|---|---|
| 기본 성공 | ≥ 20° | 첫날부터 성공 경험 제공 (초기 재활 사용자 배려) |
| 기능적 성공 | ≥ 40° | 일상생활 충분 기준 (핵심 목표) |
| 완전 ROM | ≥ 60° | 상급 목표 |

아바타 목표 호는 단일 호가 아니라 **20/40/60 눈금의 3구간 호**로 표시. 결과는 "기본 → 기능적 단계로 올라왔어요!" 진급 프레이밍.

### 6.3 UX 원칙

- 실제 판정은 0.5~1초로 하되 **UI 카운트는 3초/10초로 표시** (충분히 수행했다는 감각)
- 실패 시 "조금 더 꺾어 볼까요?" 부드러운 안내, 반복 실패 시 각도 그래프를 보여줘 자가 조정 유도
- 각도 허용 오차 ±5~10°, 시간 허용 ±0.3~0.5초

## 7. 기준값과 결과 프레이밍

### 7.1 constants.js 수치 (v2 확정)

| 항목 | 정상 ROM (스케일 끝값용) | 기능적 ROM (목표 표시용) |
|---|---|---|
| 굴곡 | 75° | 40° |
| 신전 | 70° | 40° |
| 요측 편위 | 20° | 요측+척측 합 40° |
| 척측 편위 | 35° | 〃 |

정상값은 소스별 편차(굽힘 60~90°)가 있어 중간값 계열로 확정, 출처는 주석으로. 90° 초과 측정값은 오류 판정(이상치 감지)에 사용.

### 7.2 프레이밍 원칙

- **기본 표시는 기능적 기준(40°) 대비**: "일상생활 기능 범위 87%" — 정상 대비 %는 재활 사용자에게 매일 "정상의 55%" 성적표를 주는 좌절 프레이밍이므로 금지
- 절대 각도보다 **"지난주 대비 변화 추세"** 중심 — 기술 한계·법적 포지셔닝(의료기기 아님) 양쪽 부합
- (v2) 게임 총점의 A~D 등급을 사용자에게 성적표로 노출하는 것 기각. 점수는 **내부 적응 난이도 신호로만** 사용: 낮으면 다음 세션 목표 각도 자동 하향(사용자 모르는 배려)
- (v2) 효과 문구는 정직하게: 스트레칭은 악력 강화가 아니라 **사고 예방·통증 감소** 효과 — "손목이 강해져요"(✗) → "손목을 지켜요"(✓)
- 신뢰도 낮은 상황 지속 시 "측면 촬영이 더 정확해요" 안내(강제 아님)

## 8. 콘텐츠 구조 — 숏폼 가이드 (v2 신설)

### 8.1 클립 8종

| # | 클립 | 분류 | 측정 연동 |
|---|---|---|---|
| 1 | 손 펴기·주먹 + 벌리기·모으기 | 준비 | 손가락 판정(§5.4) |
| 2 | 손목 굽힘·폄 능동 ROM | ROM | flex 측정 + 3단계 등급 |
| 3 | 좌/우 편위 | ROM | dev 측정 (**손바닥-카메라 자세**, §5.3) |
| 4 | 플렉서 스트레칭 | 스트레칭 | 자세 + 15초 유지 판정 |
| 5 | 익스텐서 스트레칭 | 스트레칭 | 〃 |
| 6 | 테이블 손바닥/손등 스트레칭 (CTS 예방) | 스트레칭 | 측정 없음(타이머만) |
| 7 | 중립 등척성 그립 | 안정성 | rawAngle 중립 체크 |

### 8.2 구조 원칙

- **클립 = 최소 단위, 루틴 = 클립 ID 배열** (예: 3분 루틴 = [1, 2, 4]) — 콘텐츠 확장이 조합으로 해결되는 데이터 구조
- 클립 내부 타임라인 공통 골격: 자세 세팅(10초) → 반복 동작(25~30초) → 최대 유지(5초) → 마무리(5~10초)
- 리듬 상수(왕복 3초×10회, 편위 왕복 2.5초 등)는 음성 큐(Web Speech API)와 가이드 애니메이션 반복 주기에 공유
- 기본 루틴 예시: 오전/오후 3분 = 준비+ROM+스트레칭 1개, 퇴근 전 5분 = 편위+테이블 스트레칭+중립 그립

## 9. 미니게임 구조 — SMoC-Wrist 4레벨 (v2 신설)

센서모터 컨트롤 재활 이론(SMoC-Wrist, J Hand Therapy 2020)의 4단계를 게임 레벨 구조로 차용. "재미용 게임"이 아니라 "임상 이론 기반 단계 설계"라는 차별점.

| 레벨 | SMoC 단계 | 게임 | 판정 |
|---|---|---|---|
| Lv1 | 고유수용감각 | "지금 내 손목이 몇 도게?" 각도 맞추기 퀴즈 | rawAngle vs 사용자 답 |
| Lv2 | 등척성 | 중립 유지 게임 (±5°, 5초 버티기, 유지 비율 점수화) | rawAngle + 패턴 B |
| Lv3 | 동적 ROM | 목표 각도 맞추기 (15°→30°→45° 연속 제시) | angle + 패턴 B (오차≤5°, 1초 80%) |
| Lv3+ | 동적 확장 | 리듬 ROM 게임 (Perfect/Good/Miss) — **조건부**: 게임 모드 한정, 목표 각도 사용자 하향 조정 가능 (타이밍 강제는 재활 모드 금지 원칙 유지) | 각도 + 타이밍 동시 판정 |
| Lv4 | 자동화 | 알림 기반 데일리 미션 ("회의 전 1분 스트레칭") — 게임이 아닌 습관 기능 | 수행 체크 |

아바타의 `draw(angle)` 인터페이스가 게임 캐릭터(손목 친구 펫 등)로 스킨 교체되는 구조 유지.

## 10. 측정 화면 구성

| 요소 | 표시 시점 | 역할 |
|---|---|---|
| 손 실루엣 가이드 | 캘리브레이션 중에만 | 위치 유도 + 색 상태 피드백, 저장 즉시 제거 |
| 동작 가이드 애니메이션 | 운동 시작 전 시범 1회 | 동작 방법 안내 (§5.3 자세 준수) |
| **실시간 아바타** | 측정 중, 구석 카드 | smoothed 각도 실시간 주입 — 사용자 움직임이 그림을 구동하는 바이오피드백 |
| 3구간 목표 호 (20/40/60) | 아바타 카드 내 | 정지된 목표 표시, 재촉 없음, 도달 시 색 변화만 |
| 각도 게이지 + 오늘 최대 | 하단 | 수치 피드백 |
| 음성 큐 | 측정 중 | 클립 리듬 상수와 동기화, 시선 분산을 소리로 해소 |

- 아바타 입력은 반드시 smoothed 값. 무효 프레임엔 마지막 자세 정지 + 회색 처리(추적 상실 정직 표시)
- 고스트 핸드(카메라 위 움직이는 실루엣 따라하기)는 기각: 속도 강제·시각 소음·정합 비용. **원칙: 정해진 애니메이션을 쫓는 구조(✗) vs 사용자 움직임이 그림을 구동하는 구조(✓)**

## 14. 구현 순서

1. **calibrate.js** — 콘솔에 L₀·W₀·θ₀·D₀가 안정적으로 찍히는 것 확인
2. 실루엣 오버레이(SVG) + 색 상태 전환
3. **measure.js** — flex 모드 → 데드존·EMA·게이트 → dev 모드
4. **judge.js** — hold 패턴 → 3단계 등급 → 프레임 비율 패턴
5. 아바타 카드 연결 (3구간 목표 호 포함)
6. storage.js — 비율 저장(2회차 무캘리브레이션), 세션 기록
7. clips.js/routines.js — 클립 2번(굽힘·폄)부터 측정 연동 검증
8. 기하 검사(2차 방어) 임계값 튜닝 — 실측 데이터 확보 후 마지막
9. (이후) 미니게임 Lv2(중립 유지)부터, Lv4 알림 미션

**보류 메모 (v2.1)**: measure.js 완성 후 **고니오미터 실측 검증 세션** 진행 — 자기 손목을 15°/30°/45°/60°로 맞추고 앱 측정값과 대조, 체계적 오차 발견 시 보정 계수 도입. 임상 도구로 정확도를 직접 검증할 수 있는 것이 이 프로젝트의 차별점.
